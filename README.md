# 🌐 AutoSec — Web Dashboard

The AutoSec Web Dashboard is a lightweight, real-time monitoring interface for visualising alerts and events generated by the AutoSec Intrusion Detection System (IDS). It connects to the IDS Alert API via HTTP/WebSockets and provides live visibility into CAN-related security events.

This README covers only the web frontend. For IDS engine, simulation, and CAN setup, see the main AutoSec repository.

⸻

✨ Features
	•	📡 Real-time alerts streamed from the IDS (WebSockets)
	•	📊 Live event feed with timestamps and severity
	•	🧭 Simple, clean UI for quick situational awareness
	•	🧾 Event logs for review during demos
	•	⚡ Low-latency updates for near real-time monitoring

⸻

🖥️ Tech Stack
	•	Frontend: HTML, CSS, JavaScript
	•	Transport: HTTP + WebSockets
	•	Hosting: Netlify
	•	Tooling: Git, VS Code

⸻

🚀 Getting Started (Web Only)

Option A — View Hosted Demo
	•	Live Dashboard: https://autosec.netlify.app

Note: Live data appears only when the IDS backend is running and streaming alerts to the API endpoint.

Option B — Run Locally
	1.	Clone the web dashboard directory:

git clone https://github.com/LuluLogics/AutoSec
cd AutoSec/web


	2.	Serve locally (any static server works):

python3 -m http.server 8080


	3.	Open in your browser:

http://localhost:8080


	4.	Configure the API endpoint (if needed) in the frontend config (e.g., config.js or environment variable) to point to your IDS Alert API.

⸻

🔌 API Integration

The dashboard expects the IDS backend to expose:
	•	WebSocket endpoint for real-time alerts
	•	HTTP endpoint (optional) for health checks / historical events

Example event schema (illustrative):

{
  "timestamp": "2026-02-09T12:34:56Z",
  "type": "CAN_FLOOD",
  "severity": "HIGH",
  "ecu": "BCM",
  "message": "Abnormal CAN rate detected"
}
